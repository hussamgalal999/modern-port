## 2026-03-31 - [CI/CD Efficiency for Documentation Workflow]
**Learning:** In projects consisting mainly of documentation and configuration, CI/CD efficiency is the primary performance surface. Using `paths-ignore` for non-code changes, `concurrency` to cancel stale runs, and `timeout-minutes` to guard resources significantly reduces waste.
**Action:** Apply these CI/CD patterns to all GitHub Actions workflows to optimize for speed and resource efficiency.
