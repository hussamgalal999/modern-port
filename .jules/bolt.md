## 2025-06-01 - CI Guards in Minimal Repositories
**Learning:** In repositories with minimal or no application source code, GitHub Actions workflows are the primary targets for performance and resource optimization. Standard guards like `concurrency`, `paths-ignore`, and `timeout-minutes` provide measurable efficiency gains by reducing redundant CI runs and preventing resource waste.
**Action:** Always check GitHub Actions workflows for missing efficiency guards in minimal repositories and implement them as a high-impact, low-risk optimization.
